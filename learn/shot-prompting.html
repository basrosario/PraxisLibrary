Foundational Technique

Shot Prompting

From zero examples to many: choosing the right amount of context for every task.

AI for Everybody Built With UD/UDL Security A+ 100%
Core Concept

Examples as Context

"Shot" in prompting refers to examples. Zero-shot means no examples, one-shot means one example, few-shot means a few, and many-shot means many. The number of examples you provide dramatically affects how well the model understands your intent.

The Central Question

How much context does the model need to understand what you want? Sometimes none. Sometimes a lot. Knowing when to use each approach is the key skill.

Quick Rule

Start with zero-shot. If results aren't right, add one example. Still not working? Add more. Don't over-engineer from the start.

The Spectrum

From Zero to Many

Each level serves different purposes and task types

Zero-Shot No Examples
What it is:
No examples provided. Relies entirely on clear instructions and the model's pre-trained knowledge.
Example prompt:
"Translate the following English text to French: 'Hello, how are you today?'"
Best for simple, well-defined tasks the model already knows how to do
When to Use
Ideal scenarios:
  • Standard tasks (translation, summarization, Q&A)
  • Clear, unambiguous instructions
  • Tasks where format is obvious
  • When you want speed over precision
Advantages:
  • Fastest to write
  • Uses least tokens
  • Good baseline to test from
One-Shot 1 Example
What it is:
A single example demonstrates the expected input-output pattern before your actual request.
Example prompt:
"Convert to title case:

Input: the quick brown fox
Output: The Quick Brown Fox

Input: hello world from AI
Output:"
Best for format-sensitive tasks and custom output structures
When to Use
Ideal scenarios:
  • Custom formatting requirements
  • Specific output structure needed
  • Pattern is clear from one example
  • Balancing context with efficiency
Advantages:
  • Shows exact format expected
  • Minimal token overhead
  • Often sufficient for many tasks
Few-Shot 2-5 Examples
What it is:
Multiple examples show variation and reinforce the pattern, helping the model understand nuances and edge cases.
Example prompt:
"Classify the sentiment:

Text: I love this product!
Sentiment: Positive

Text: Worst purchase ever.
Sentiment: Negative

Text: It works as expected.
Sentiment: Neutral

Text: Amazing quality, highly recommend!
Sentiment:"
Best for complex reasoning, edge cases, and establishing clear patterns
When to Use
Ideal scenarios:
  • Classification tasks with multiple categories
  • Complex transformations
  • Tasks requiring reasoning
  • When edge cases matter
Best practices:
  • Cover different scenarios
  • Include edge cases if relevant
  • Maintain consistent formatting
  • 3-5 examples is usually optimal
Many-Shot 10+ Examples
What it is:
Extensive examples for highly specialized or unusual tasks. Approaches in-context "fine-tuning" by providing substantial demonstration data.
Use case:
A company's specific email classification system with 15+ examples covering all their custom categories, edge cases, and priority rules.
Best for domain-specific patterns, proprietary formats, and unusual tasks
When to Use
Ideal scenarios:
  • Proprietary classification systems
  • Company-specific terminology
  • Highly specialized domains
  • Tasks the model has never seen
Considerations:
  • Higher token cost
  • May hit context limits
  • Diminishing returns after ~20 examples
  • Consider fine-tuning for production
Decision Guide

Choosing the Right Approach

Start Simple

Always begin with zero-shot. It's the fastest to test and often works for standard tasks. Only add examples when results don't match expectations.

First Step

Add Minimally

When zero-shot fails, add just one example first. Many tasks only need a single demonstration to click into place.

Efficiency

Cover Variation

If one example isn't enough, add examples that show different scenariosâ€”not just more of the same. Diversity beats repetition.

Quality

Know the Limits

More examples have diminishing returns. If 5 examples don't work, 20 probably won't either. Consider if the task needs a different approach.
