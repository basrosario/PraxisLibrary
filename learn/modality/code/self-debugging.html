Self-Debugging

AI-assisted bug identification, explanation, and fixing.

The Concept

Debugging with AI

Self-Debugging is a prompting technique where AI models identify, explain, and fix bugs in code. This approach leverages the model's ability to trace logic, interpret errors, and reason about expected vs actual behavior—turning debugging from a hunt into a conversation.

Core Insight

Models can often spot bugs they would have created. Re-examining code for errors activates different reasoning patterns than initial generation.

The Workflow

1) Generate code. 2) Test and collect errors. 3) Debug with feedback. 4) Apply fix. 5) Verify and iterate.

Strategies

Debugging Approaches

Approach

Start with an error message and work backward to find the root cause.

Best For

Runtime errors, exceptions, compile errors, type mismatches. Provide the error message, code, and ask for explanation and fix.

Approach

Walk through code execution step-by-step to find where behavior diverges from expectations.

Best For

Logic errors, wrong output, off-by-one errors, loop issues. Ask to trace variable values through execution.

Approach

Have the model explain what code does line-by-line. Bugs often become apparent during explanation.

Best For

Understanding unfamiliar code, subtle logic errors, code you didn't write. State the intended purpose for comparison.

Approach

Provide failing test cases and have the model fix code to pass them.

Best For

Clearly defined expected behavior, regression bugs, edge case failures. Include input, expected output, and actual output.

Bug Types

Common Bug Patterns

Off-by-One

Loop bounds, array indexing, and range calculations. Always verify boundary conditions.

Null Handling

Generated code often assumes values exist. Check for null/undefined on optional data.

Type Mismatches

Implicit conversions in loosely-typed languages cause unexpected behavior.

Async Issues

Race conditions, missing awaits, callback ordering problems in asynchronous code.

Techniques

Debug Prompt Patterns

The Rubber Duck

"Act as my rubber duck: explain what each section does as if teaching a junior developer."

Adversarial Reviewer

"Review as a skeptical code reviewer. Assume there IS a bug somewhere. What could go wrong?"

Hypothesis Tester

"I think the bug is in the sorting logic. Trace through with [input] and tell me if I'm right."

Minimal Fix

"Make the MINIMAL change necessary. Don't refactor—show only the lines that need to change."
