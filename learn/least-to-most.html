Decomposition Technique

Least-to-Most Prompting

Conquer complexity by starting small. Break problems into sub-problems, solve them in order, and build to the answer.

The Approach

Divide and Conquer for AI

Least-to-Most prompting handles problems that are too complex to solve in one step. First, decompose the problem into simpler sub-problems. Then solve each sub-problem, using earlier answers to inform later ones.

Why It Works

By solving easier problems first, the model builds context and intermediate results that make harder problems tractable. Each answer becomes a stepping stone.

Two-Stage Process

First decompose into subproblems, then solve from simplest to most complex. Each solution builds on the last.

The Process

Two-Stage Framework

Decomposition followed by sequential solving

Stage 1: Decomposition

Break the complex problem into a sequence of simpler sub-problems. Order them from easiest to hardest, or in logical dependency order.

Prompt: "What sub-problems need to be solved first?"

Stage 2a: Solve First Sub-Problem

Solve the simplest or foundational sub-problem. This establishes a base of knowledge for subsequent steps.

Start with what you can answer directly

Stage 2b: Solve Next Sub-Problem

Using the answer from the previous step, solve the next sub-problem. The earlier answer provides context.

Build on established answers

Stage 2c: Continue Until Complete

Repeat until all sub-problems are solved. The final sub-problem's answer is typically the answer to the original complex problem.

Final answer emerges from the sequence
In Action

Least-to-Most Example

See how a complex word problem becomes manageable

All At Once Overwhelmed
Problem: Amy has 4 times as many apples as Ben. Ben has 3 more apples than Carl. Carl has 2 apples. How many apples does Amy have?
Direct attempt: The model must track multiple relationships simultaneously, often losing track or making arithmetic errors in the chain.
Too many relationships to track at once
What Happens Error-Prone
Common Errors:
- Confuses who has more than whom
- Applies multiplier at wrong step
- Loses track of intermediate values
Result: Wrong answer or inconsistent reasoning
Cognitive overload leads to mistakes
Stage 1: Decompose Clear Steps
Sub-problems identified:
1. How many apples does Carl have?
2. How many apples does Ben have?
3. How many apples does Amy have?
Order: Carl → Ben → Amy (dependency chain)
Clear dependency ordering
Stage 2: Solve Sequentially Step by Step
Solving each:
Q1: Carl has 2 apples (given)
Q2: Ben = Carl + 3 = 2 + 3 = 5 apples
Q3: Amy = Ben × 4 = 5 × 4 = 20 apples
Final Answer: Amy has 20 apples
Each step builds on previous, verifiable
Applications

When Least-to-Most Excels

Multi-Step Word Problems

Math problems with chained relationships where each value depends on previous calculations.

Compositional Tasks

Commands like "turn left twice then jump" that combine multiple primitive operations.

Complex Debugging

Finding bugs by first understanding the expected behavior, then tracing actual behavior, then identifying the discrepancy.

Reading Comprehension

Questions requiring synthesis from multiple passages, where understanding earlier parts informs later understanding.

Comparison

Least-to-Most vs. Chain-of-Thought

Chain-of-Thought

Generates reasoning steps linearly. Good for problems where the path forward is clear. May struggle when the problem structure isn't immediately obvious.

Linear reasoning

Least-to-Most

Explicitly decomposes first, then solves. Better for problems requiring structural understanding. Two-stage approach ensures problem structure is clear before solving.

Structured decomposition
When to Choose Least-to-Most

Use Least-to-Most when: (1) The problem has clear sub-problems with dependencies, (2) Chain-of-Thought loses track mid-reasoning, (3) You need to explain or verify each step independently.

Related Frameworks

Other decomposition approaches

Foundation Framework Chain-of-Thought Linear step-by-step reasoning for problems with clear progression.
Related Technique Prompt Chaining Connect multiple prompts where each output feeds the next input.
Enhancement Self-Consistency Apply multiple reasoning paths and vote on the answer for verification.
