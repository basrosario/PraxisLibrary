Self-Correction

CRITIC

Critique and revise outputs with external tool verification.

The Concept

Tool-Augmented Self-Critique

CRITIC (Self-Correcting with Tool-Interactive Critiquing) enables LLMs to verify their outputs using external tools like search engines, code interpreters, or calculators. Rather than relying solely on internal knowledge, the model actively checks its claims against external sources.

Key Innovation

External tools provide ground truth that the model's internal knowledge may lack. This allows correction of factual errors, computational mistakes, and outdated information.

The Loop

1) Generate initial output. 2) Identify claims to verify. 3) Use tools to check. 4) Critique based on results. 5) Revise if needed.

Tools

Verification Methods

Search Engine

Verify factual claims, dates, names, and current events against web sources.

Code Interpreter

Execute code to verify calculations, algorithms, and logical operations.

Knowledge Base

Query structured databases for entity relationships and verified data.

Example

CRITIC in Action

Initial Output

"The Eiffel Tower was completed in 1887 and stands 324 meters tall."

CRITIC Process

Tool Query: "Eiffel Tower completion date"

Tool Result: Completed in 1889

Critique: Completion date is incorrect (1887 vs 1889)

Revised: "The Eiffel Tower was completed in 1889 and stands 324 meters tall."
