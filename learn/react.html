The ReAct Framework

Reasoning and Acting. Make AI thinking transparent and verifiable by alternating between thought, action, and observation in an iterative loop.

The ReAct Cycle

An iterative loop that builds understanding through transparent reasoning

ReAct Loop
Thought Analyze the situation
Processing Thought

Action Take a specific step

The Three Elements

Each element serves a distinct purpose in the reasoning process

1

Thought

Analyze the current state and decide what to do next. This is where the AI shows its reasoning—what it knows, what it needs to find out, and what the logical next step is.

Example Thoughts:
  • "The error is intermittent, suggesting a resource issue rather than code logic..."
  • "Given that the timeouts correlate with peak hours, this points to..."
  • "I need to verify whether the database or external API is the bottleneck..."
2

Action

Take a specific action based on the reasoning. This might be gathering information, making a calculation, testing a hypothesis, or applying a solution.

Example Actions:
  • "Check the error logs for timestamps and patterns"
  • "Examine the stack traces to identify which service is timing out"
  • "Compare response times between database and API calls"
3

Observation

Note the result of the action. What did we learn? Does this confirm or challenge our hypothesis? This feeds into the next Thought.

Example Observations:
  • "Errors spike between 9-10 AM and 2-3 PM, corresponding to peak usage"
  • "The timeouts are occurring on payment processor API calls, not database"
  • "Response times increase 5x when concurrent requests exceed 100"

ReAct in Action

A complete debugging scenario showing multiple iterations

Problem Statement

"Our API is returning intermittent 500 errors. Can you help me debug this?"

Iteration 1
Thought

Intermittent 500 errors suggest the issue isn't with the code logic itself (which would fail consistently), but likely related to resources, connections, or external dependencies.

Action

Check the error logs for timestamps and look for patterns. Are errors clustered at certain times?

Observation

Errors spike between 9-10 AM and 2-3 PM, which corresponds to peak usage times. The errors include "connection timeout" messages.

Iteration 2
Thought

Connection timeouts during peak usage suggests either database connection pool exhaustion or rate-limiting from an external service.

Action

Examine the stack traces to identify which service call is timing out—database or external API?

Observation

The timeouts are occurring on calls to our payment processor API, not the database.

Conclusion

Root Cause Identified

The payment processor is rate-limiting our requests during peak times.

Recommendations:
  • Implement request queuing to smooth out peak demand
  • Consider upgrading the API tier for higher rate limits
  • Add retry logic with exponential backoff

Why This Works

Notice how each step builds on the previous. The AI could have jumped straight to "check your payment processor," but by showing its reasoning, you can verify the logic is sound. If you knew the timeouts started before peak hours, you could interject and redirect the investigation.

When to Use ReAct

Best for problems requiring transparency and iteration

Debugging & Troubleshooting

Work through issues systematically, testing hypotheses and narrowing down causes with visible reasoning.

Research & Analysis

Explore topics by gathering information, synthesizing findings, and drawing conclusions step by step.

Decision Making

Evaluate options, consider trade-offs, and arrive at a reasoned recommendation with full transparency.

Complex Calculations

Break down multi-step problems, showing work at each stage so errors can be caught early.

Planning & Strategy

Consider constraints, evaluate approaches, and build a step-by-step plan with justified choices.

Code Review

Systematically analyze code, explain reasoning, and provide feedback with clear justification.

When NOT to Use ReAct

For simple, direct tasks—like writing a quick email or answering a factual question—ReAct adds unnecessary overhead. Use it when the problem is complex enough that you'd benefit from seeing the AI's reasoning, or when errors would be costly to fix after the fact.

How to Prompt for ReAct

You can request step-by-step reasoning explicitly or naturally

Explicit

Named Framework Request

Use the ReAct framework to help me decide whether to build or buy a CRM system for my small business.

For each step:

  1. State your thought process
  2. Take an action (analyze, compare, research)
  3. Observe the result
  4. Repeat until you reach a conclusion
=
Natural

Conversational Request

I'm trying to decide whether to build a custom CRM or buy an existing solution for my small business.

Can you think through this with me step by step? Walk me through your reasoning, consider the trade-offs, and show your work so I can follow your logic.

I want to understand how you arrive at your recommendation.

Natural Phrases That Trigger ReAct Responses

"Think through this step by step" "Show your reasoning" "Walk me through your logic" "Explain how you arrived at that" "Let me follow your thought process" "Work through this with me"
