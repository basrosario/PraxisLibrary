Memory Systems

Memory of Thought

Every problem you solve teaches you something about the next one. Memory of Thought captures successful reasoning chains and retrieves them when facing similar challenges — building a personal library of proven approaches that grows smarter with every use.

Framework Context: 2023

Introduced: Memory of Thought was published in 2023 by Li et al. The technique introduces a two-phase process: during a pre-thinking phase, the model solves problems and stores its unlabeled reasoning chains in a memory bank. During the test phase, the system retrieves the most relevant stored reasoning chains as demonstrations for new problems. Unlike static few-shot prompting that uses the same examples every time, MoT builds a dynamic memory bank that grows over time — each successfully solved problem enriches the library of available reasoning patterns.

Modern LLM Status: Persistent memory is increasingly built into AI systems. MoT’s key insight — that past reasoning chains (not just answers) are valuable — has influenced how modern systems leverage conversation history. Claude’s Projects, GPT’s memory features, and custom instruction systems all reflect the principle that accumulated context improves future performance. The technique remains most relevant for production pipelines where reasoning quality compounds over time.

The Core Insight

Your Past Reasoning Is Your Best Teacher

Think of a student reviewing worked solutions before an exam. They don’t just memorize answers — they study the reasoning process that led to each solution. Memory of Thought applies this same principle to AI: the model builds a library of successful reasoning approaches, indexed by problem type, that it can draw upon when facing new challenges.

The power is in the chains, not the answers. A stored answer tells you “42.” A stored reasoning chain tells you how to arrive at 42 — the decomposition strategy, the intermediate steps, the verification logic. When retrieved for a similar problem, that chain serves as a worked example that guides the model through an analogous reasoning path.

Over time, the memory bank becomes a curated collection of domain-specific reasoning strategies — an ever-growing playbook that makes the system more capable with every problem it solves.

Why Chains Matter More Than Answers

Storing just the final answer to a past problem gives you one data point. Storing the full reasoning chain gives you a reusable template — the decomposition strategy, intermediate calculations, verification steps, and logical structure that can transfer to entirely new problems. When the model retrieves a chain for a similar problem, it gains not just what to think, but how to think about that class of problem.

The Memory of Thought Process

Four stages from solving problems to building reusable reasoning memory

1

Solve and Store

The model solves problems and saves the full reasoning chain — not just the final answer — to a persistent memory bank. Each entry captures the complete thought process: how the problem was decomposed, what intermediate steps were taken, and how the final answer was derived.

Example

Solving “Calculate compound interest on $5,000 at 4% for 3 years” — the full chain (formula identification, substitution, step-by-step calculation, verification) is stored alongside the problem description.

2

Index and Organize

Stored chains are categorized by problem type, domain, and reasoning strategy. This indexing enables efficient retrieval later — the system can quickly find relevant past reasoning based on structural similarity rather than surface-level keyword matching.

Example

The compound interest chain is tagged as “financial calculation / exponential growth / multi-step arithmetic” — making it retrievable for any problem with similar mathematical structure.

3

Retrieve

When a new problem arrives, the system searches the memory bank for the most similar past reasoning chains. Similarity is measured not just by topic but by problem structure — a depreciation calculation might retrieve the compound interest chain because both involve exponential formulas applied over time periods.

Example

New problem: “Calculate equipment depreciation of $12,000 at 15% over 5 years.” The system retrieves the compound interest chain as a structurally similar worked example.

4

Apply and Augment

The retrieved chains serve as demonstrations that guide the model through the new problem. After solving it, the new reasoning chain is added to the memory bank — growing the library and making the system more capable for future problems. The memory bank continuously improves.

Example

Using the compound interest chain as a template, the model adapts the approach for depreciation — then stores this new depreciation chain, enriching the memory for future financial calculations.

See the Difference

Why dynamic memory outperforms static examples

Static Few-Shot

Approach

Use the same 3 hand-picked examples for every problem, regardless of domain or complexity. The examples never change and were selected once during prompt design.

Result

Examples may be irrelevant to the current problem. No learning from past successes. No adaptation to new domains. Performance plateaus after initial setup.

Same examples every time, no learning, no adaptation
VS

Memory of Thought

Approach

Dynamically retrieve the most relevant past reasoning chains for each new problem. The example bank grows with every solved problem, automatically specializing to your domain.

Result

Every problem gets the most structurally relevant demonstrations. The system learns from its own successes. Performance improves over time as the memory bank deepens.

Dynamic example bank that grows, adapts, and specializes
