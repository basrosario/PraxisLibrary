Responsible AI Policy

A formal declaration of how Praxis Library governs, monitors, and ensures the responsible use of artificial intelligence across every aspect of this platform.

AI for Everybody Built With UD/UDL Security A+ 100% Performance 100% AI Assisted Building Claude Code Community GitHub

An organization that educates the public on responsible AI practices has an obligation to uphold those same standards within its own operations. This policy documents the principles, governance mechanisms, risk management procedures, regulatory alignment, and continuous monitoring processes that govern how Praxis Library develops, deploys, and maintains AI-assisted educational content.

This policy applies to all AI systems used in the creation, maintenance, and delivery of Praxis Library content and tooling. It is reviewed and updated with each significant change to our AI governance practices.

Responsible AI Principles

Six foundational commitments that govern every decision at Praxis Library

1. Transparency & Disclosure

Every instance of AI involvement in the creation of Praxis Library content is disclosed. Every git commit includes explicit AI co-authorship attribution. Every page displays an AI-assisted building badge. The ethics ticker delivers continuous transparency reminders across the platform. We do not obscure, minimize, or omit the role of AI in any aspect of our operations.

Evidence: AI Assisted Building • Co-Authored-By attribution in all commits • Site-wide ethics ticker

2. Human Oversight & Accountability

No AI-generated output is published without human review and approval. Human judgment governs all strategic, editorial, and architectural decisions. AI serves as an implementation tool under direct human supervision. The human operator retains full accountability for all published content, regardless of whether AI assisted in its creation.

Evidence: Collaboration Model • Human review documented in project governance files

3. Privacy by Architecture

Praxis Library achieves data privacy not through policy constraints on data collection, but through architectural decisions that eliminate data collection entirely. Zero cookies. Zero analytics. Zero tracking. Zero telemetry. All seven interactive tools process data exclusively within the user’s browser. No personal information is transmitted to any server at any time.

Evidence: Data Retention Policy • Security Policy

4. Accessibility & Inclusivity

AI literacy is a civic necessity. Praxis Library is committed to ensuring that no individual is excluded from accessing this educational resource by ability, neurodivergence, economic status, or technical background. The platform maintains WCAG AA compliance, provides dedicated neurodivergent learning resources, charges no fees, and requires no account creation.

Evidence: AI for Everybody • Universal Design • ND Hub

5. Verification & Accuracy

All factual claims published on Praxis Library are verified against authoritative sources. Citation standards require .gov and .edu domains with publication dates between 2024 and 2026. The VERIFY framework used to teach users critical evaluation of AI outputs is applied internally to all AI-assisted content before publication. An automated 12-category audit tool enforces these standards programmatically.

Evidence: AI Safety & Ethics • Automated site audit tool • Citation sourcing standards

6. Open Source & Community Governance

The complete source code, content, development history, and governance documentation for Praxis Library are publicly available on GitHub. This enables external review, community contribution, and independent verification of all claims made in this policy. Transparency is not a statement—it is a verifiable condition of our codebase.

Evidence: GitHub Repository • Full commit history with AI attribution

AI Systems Documentation

Formal inventory of all AI systems used in the development and delivery of Praxis Library

Development

Claude Code (Anthropic)

Purpose: Code generation, content drafting, refactoring, cross-file updates, and architecture implementation under direct human supervision.

Model: Claude (current production model, updated as Anthropic releases new versions).

Scope: All AI-generated output undergoes mandatory human review before publication. No autonomous publishing capability exists.

Data Flow: Local development environment only. No user data is processed by Claude. The AI operates on project files stored on the developer’s machine.

Known Limitations: May produce hallucinated content, requires factual verification, cannot access live web resources, subject to training data biases.

Quality Assurance

Automated Site Audit Tool

Purpose: Programmatic enforcement of quality, security, accessibility, and citation standards across all 165 HTML files.

Type: Deterministic Python script (not an AI model). Executes rule-based checks with zero probabilistic output.

Scope: 12 audit categories including CSP compliance, citation freshness, domain validation, broken links, accessibility attributes, and content consistency.

Frequency: Executed before every deployment and after any significant content changes.

User-Facing

Interactive Educational Tools (7)

Purpose: Prompt Analyzer, Framework Finder, Preflight Checklist, Prompt Builder, Persona Architect, Hallucination Spotter, and Readiness Quiz.

Processing: 100% client-side execution within the user’s browser. Zero server communication. Zero data transmission.

Data Handling: No user input is stored, transmitted, logged, or retained in any form. All processing terminates when the browser tab closes.

Declaration

No Additional AI Systems

Praxis Library does not employ any additional AI systems beyond those documented above. Specifically, Praxis Library does not use:

  • Analytics or behavioral tracking AI
  • Recommendation or personalization engines
  • Automated content moderation systems
  • Advertising or targeting algorithms
  • A/B testing or optimization platforms
  • User profiling or segmentation tools

Risk Self-Assessment

Identified risks associated with Praxis Library’s use of AI, with documented mitigation strategies

Assessment Scope

This self-assessment evaluates risks arising from the use of AI in the development and maintenance of Praxis Library. It is reviewed with each significant change to AI governance practices and updated in the Transparency Changelog below.

Risk Category

Content Accuracy & Hallucination

Risk: AI-assisted content generation may introduce factual errors, fabricated citations, or hallucinated technical information into educational materials relied upon by the public.

Likelihood: Moderate (inherent to all large language model outputs).

Mitigation Controls:

  • Mandatory human review of all AI-generated content before publication
  • Citation sourcing standards requiring .gov and .edu domains with 2024–2026 publication dates
  • Automated site audit tool with citation freshness and domain validation checks
  • Internal application of the VERIFY framework to all factual claims
  • Historical context notices on all framework pages to flag evolving information
Residual Risk Level: Low — multiple overlapping controls reduce hallucination exposure to an operationally acceptable level.
Risk Category

Bias in AI-Generated Content

Risk: The AI model used for content generation may reflect biases present in its training data, potentially producing content that is exclusionary, culturally insensitive, or inequitable.

Likelihood: Moderate (systemic in all language models).

Mitigation Controls:

  • Human editorial review for inclusive language and diverse representation
  • Dedicated neurodivergent learning resources developed with ND-informed design principles
  • Universal Design (UD/UDL) framework applied across all content
  • Open-source codebase enabling community review and bias identification
  • Prompt examples designed to model equitable and responsible AI use
Residual Risk Level: Low — editorial oversight and community review provide effective bias detection.
Risk Category

AI Provider Dependency

Risk: Praxis Library relies on Anthropic’s Claude Code for AI-assisted development, creating a single-provider dependency that could affect continuity of operations.

Likelihood: Low (Anthropic maintains stable commercial operations).

Mitigation Controls:

  • Praxis Library is a static website—no runtime AI dependency exists in production
  • All AI output is committed as plain HTML, CSS, and JavaScript files
  • The platform would continue to function indefinitely without any AI provider
  • Development methodology is AI-model-agnostic and could transition to any capable code assistant
Residual Risk Level: Minimal — static architecture eliminates runtime dependency entirely.
Risk Category

Content Currency & Obsolescence

Risk: AI models have training data cutoff dates. Content generated with AI assistance may become outdated as the AI field evolves rapidly.

Likelihood: Moderate (AI landscape changes continuously).

Mitigation Controls:

  • Citation freshness policy enforcing 2024–2026 publication dates only
  • Historical context notices on all framework pages acknowledging the evolving nature of techniques
  • Regular content review cycles integrated into the development workflow
  • Automated audit tool flags pre-2024 citations as ERROR-level violations
Residual Risk Level: Moderate — inherent to any educational resource covering a rapidly evolving field. Mitigated through active maintenance.

Continuous Monitoring & Evaluation

Systematic processes that verify ongoing compliance with the standards documented in this policy

Automated Site Audit

A 12-category Python audit tool performs 100+ automated checks across all HTML files before every deployment, covering security, accessibility, citations, broken links, and content consistency.

Citation Freshness Enforcement

All external citations are programmatically validated to ensure they reference .gov or .edu domains with publication dates between 2024 and 2026. Pre-2024 sources trigger ERROR-level audit violations.

Security Scanning

Content Security Policy (CSP) A+ rating verified continuously. Zero inline scripts, zero inline styles, zero external resource dependencies enforced by both audit tool and deployment architecture.

Accessibility Verification

WCAG AA compliance verified across all pages: language attributes, meaningful alt text, heading hierarchy, keyboard navigation, and 4.5:1 color contrast ratios audited programmatically.

Content Consistency Checks

Automated cross-referencing of filesystem page counts, glossary term counts, tool inventories, and navigation links against documented values to detect drift between claims and reality.

Community Review & Feedback

Open-source repository on GitHub enables continuous community review. Issues, pull requests, and discussions provide a public forum for identifying concerns, suggesting improvements, and reporting incidents.

Incident Reporting & Response

Procedures for reporting concerns related to AI governance, content accuracy, bias, or safety

Praxis Library maintains clear pathways for reporting concerns. All reports are publicly visible to ensure transparency and accountability in our response process.

GitHub Issues (Primary)

Open a new issue on the Praxis Library repository. Use the label rai-concern for responsible AI matters. All issues are publicly visible and tracked to resolution.

Pull Requests

If you can resolve the issue directly, submit a pull request with the fix. Community contributions are reviewed and, where appropriate, incorporated into the platform.

Content Accuracy Reports

If you identify factual errors, hallucinated citations, outdated information, or biased content, report it via GitHub Issues with the label content-accuracy. Citation integrity is a priority concern.

Response Commitment

Praxis Library commits to acknowledging all RAI-related reports within 7 days and publishing a response on the issue thread. For content accuracy concerns involving active misinformation, we prioritize same-day review. All response actions are documented in the public issue history.
