Severity Distribution

Issues grouped by severity level across all categories.

--
Errors
Errors are confirmed violations of enforced standards that directly compromise security, accessibility, or content integrity. Examples include inline scripts breaking CSP A+ compliance, stale citations older than 2024, and unreachable source URLs. Every error must be resolved before the site is considered production-ready.
--
Warnings
Warnings flag best-practice deviations that do not fail the audit but signal quality gaps. Common examples include highlight boxes missing source attribution, deprecated component patterns, and links without date-tracking attributes. Each warning includes a specific fix suggestion to prevent it from becoming an error in future updates.
--
Info
Info items are inventory records and transparency notes, not problems to fix. They document per-page citation counts, source status, bot-blocked URLs, and bias term coverage so nothing is hidden or silently skipped.
--
Verified Repository
Items confirmed by human review with screenshot proof stored in assets/Citation images/. Each entry is registered in Audit/verified-items.json and the proof file must exist on disk for the verification to hold.
